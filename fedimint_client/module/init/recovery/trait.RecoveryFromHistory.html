fn finalize_dbtx<'life0, 'life1, 'life2, 'life3, 'async_trait>( &'life0 self, dbtx: &'life1 mut ClientDbTxContext<'life2, 'life3, <Self::Init as ClientModuleInit>::Module> ) -> Pin<Box<dyn Future<Output = Result<()>> + Send + 'async_trait>>where Self: 'async_trait, 'life0: 'async_trait, 'life1: 'async_trait, 'life2: 'async_trait, 'life3: 'async_trait,

Finalize the recovery converting the tracked state to to final changes in the database.

This is the only place during recovery where module gets a chance to create state machines, etc. and that’s why dbtx is of ClientDbTxContext type.

Notably this function is running in a database-autocommit wrapper, so might be called again on database commit failure.
