Uses the in-memory config to write a config backup tar archive that guardians can download. Private keys are encrypted with the guardian password, so it should be safe to store anywhere, this also means the backup is useless without the password.

Changes the guardian password by re-encrypting the private config and changing the on-disk password file if present. fedimintd is shut down afterward, the user’s service manager (e.g. systemd is expected to restart it).
