fn all_members(&self) -> &BTreeSet<PeerId, Global>

List of all federation members for the purpose of iterating each member in the federation.

The underlying implementation is resonsible for knowing how many and PeerIds of each. The caller of this interface most probably have some idea as well, but passing this set across every API call to the federation would be inconvenient.
